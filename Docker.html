                                                            • 一、基本介绍

                                                            （一）概述

                                                            理念： Build, Ship and Run Any App, Anywhere（随时随地构建、发布和运行任何应用程序）

                                                             

                                                            结构：

                                                             

                                                            （二）优点

                                                            1. 系统平滑移植，容器虚拟化技术（打包所有的源码、配置、环境、版本等信息。一次打包，处处运行）
                                                            2. 容器之间的运行环境相互隔离
                                                            3. container复用了本主机的操作系统，仅包含runTime的最小要求，无需捆绑一整套操作系统

                                                            3.资源管理

                                                            docker system df

                                                            • 作用：查看镜像/容器/数据卷所占的空间

                                                            • 参数说明：

                                                              • -v：显示详细信息

                                                            • 回显说明：

                                                              • RECLAIMABLE：可回收空间（未被使用的资源）
                                                              • SHARED SIZE：共享层大小
                                                              • UNIQUE SIZE：独享层大小
                                                              • CONTAINERS：启动容器数
                                                              • LOCAL VOLUMES：挂载的本地卷数
                                                              • LINKS：使用的所有容器数（包括停止的）

                                                             

                                                            docker system info

                                                            • 作用：显示Docker系统的信息（等同 docker info）

                                                             

                                                            docker system prune

                                                            • 作用：清除没有使用到的容器/镜像/网络

                                                             

                                                            （二）镜像命令

                                                            docker images

                                                            • 作用：列出本地主机上的镜像列表

                                                            • 参数说明：

                                                              • -a：列出本地所有的镜像（含临时映像层）
                                                              • -q：只显示镜像ID

                                                            • 回显说明：

                                                              • REPOSITORY：镜像的仓库源和名称
                                                              • TAG：镜像的标签版本号
                                                              • IMAGE ID：镜像ID
                                                              • CREATED：镜像创建时间
                                                              • SIZE：镜像大小

                                                             

                                                            docker search {imageName}

                                                            • 作用：查找远程库中的镜像文件（远程仓库地址）

                                                            • 参数说明：

                                                              • --limit {n}：限制返回数，默认25

                                                            • 回显说明：

                                                              • NAME：镜像名称
                                                              • DESCRIPTION：镜像说明
                                                              • STARS：点赞数量
                                                              • OFFICIAL：是否官方镜像
                                                              • AUTOMATED：是否自动构建（Docker Hub自动build 或者 手动输入指令build）

                                                            • 示例：

                                                              • docker search --limit 5 redis：查看远程库中的redis镜像，并显示前5个结果

                                                             

                                                            docker pull {imgName}[:imgTag]

                                                            • 作用：拉取镜像，Tag默认为latest

                                                             

                                                            docker push {imgName}[:imgTag]

                                                            • 作用：推送镜像，Tag默认为latest

                                                             

                                                            docker rmi {image/ID}

                                                            • 作用：删除单个/多个镜像

                                                            • 参数说明：

                                                              • -f：强制删除，并删除关联的容器实例
                                                              • --no-prune：不删除‘未使用层’

                                                            • 示例：

                                                              • docker rmi -f $(docker images -qa)：删除全部镜像

                                                             

                                                            docker save

                                                            • 作用：将一个/多个镜像打包成tar (建议使用name+tag, 而不是ID, 后者导出为虚悬镜像)

                                                            • 参数说明：

                                                              • -o：指定tar包

                                                             

                                                            docker load

                                                            • 作用：将tar包中加载镜像

                                                            • 参数说明：

                                                              • -i：指定加载的tar包

                                                             

                                                            docker build

                                                            • 作用：从 Dockerfile 构建镜像

                                                            • 参数说明：

                                                              • -t：指定 name:tag
                                                              • -f：指定Dockerfile文件

                                                            • 示例：

                                                              • docker build -t grafana:5.4.3-zh .

                                                             

                                                             

                                                            （三）容器命令

                                                            1.启动进入

                                                            docker run

                                                            • 作用：创建容器

                                                            • 参数说明：

                                                              • --name={containerName}：设置容器名称
                                                              • -it：启动交互式容器
                                                              • -d：后台启动
                                                              • -v {hostPath:containerPath}：文件挂载
                                                              • -p {hostPort:containerPort}：端口映射
                                                              • --netword={network}：网络设置
                                                              • -e {evnKey:evnVal}：设置环境变量
                                                              • --env-file={filePath}：从文件中加载环境变量
                                                              • --cpus={num}：限制CPU使用核数
                                                              • --memory={size}：限制内存使用（例：512m/1g）
                                                              • --rm：容器退出后自动删除（适用测试情况）

                                                             

                                                            docker attach {containerId} TODO ？？？未发现使用场景

                                                            • 作用：将本地标准输入(stdin)、输出(stdout)和错误(stderr)流连接到正在运行的容器

                                                             

                                                            docker exec {containerId} {command}

                                                            • 作用：在正在运行的容器中执行新命令

                                                            • 参数说明：

                                                              • -i：保持输入打开
                                                              • -t：分配‘伪终端’
                                                              • -u：指定执行命令的用户
                                                              • -w：指定执行命令的工作目录
                                                              • -d：后台执行

                                                             

                                                            2.生命周期

                                                            docker start {containerID}

                                                            • 作用：启动一个或多个已停止的容器

                                                             

                                                            docker restart {containerID}

                                                            • 作用：重新启动一个或多个容器

                                                             

                                                            docker stop {containerID}

                                                            • 作用：停止一个或多个正在运行的容器（优雅停止）

                                                             

                                                            docker pause{containerID}

                                                            • 作用：暂停一个或多个正在运行的容器
                                                            • 说明：容器状态变成Paused，仅释放容器的CPU资源。可用于快速恢复

                                                             

                                                            docker unpause{containerID}

                                                            • 作用：恢复一个或多个正在运行的容器

                                                             

                                                            docker kill {containerID}

                                                            • 作用：停止一个或多个正在运行的容器（立即强制停止）

                                                             

                                                            docker rm {containerID}

                                                            • 作用：删除一个或多个容器

                                                            • 参数说明：

                                                              • -f：强制删除运行中的容器
                                                              • -v：同时删除‘匿名卷’

                                                             

                                                            3.信息查看

                                                            docker ps

                                                            • 作用：查看正在运行的容器

                                                            • 参数说明：

                                                              • -a：显示运行中和停止状态的所有容器
                                                              • -l：显示最近创建的容器
                                                              • -n {N}：显示最近创建的N个容器
                                                              • -q：只显示容器编号
                                                              • -s：显示容器大小

                                                             

                                                            docker logs {containerID}

                                                            • 作用：查看容器日志

                                                            • 参数说明：

                                                              • -f：实时跟踪日志输出
                                                              • -n {N}：显示最后N行日志；默认显示全部日志
                                                              • -t：显示日志的时间戳
                                                              • --since {time}：显示从指定时间开始的日志
                                                              • --until {time}：显示指定时间之前的日志

                                                            • 使用示例：

                                                              • docker logs --since 2023-01-01T00:00:00Z my_container docker logs --since 2h my_container 最近2小时的日志
                                                              • docker logs --until 2023-01-01T00:00:00Z my_container docker logs --until 30m my_container 30分钟前的日志

                                                             

                                                            docker top {containerID}

                                                            • 作用：查看容器内运行的进程

                                                            • 回显说明：

                                                              • UID：进程所属用户ID
                                                              • PID：进程ID
                                                              • PPID：父进程ID
                                                              • C：CPU使用百分比
                                                              • STIME：进程启动时间
                                                              • TTY：进程关联终端
                                                              • TIME：进程使用CPU总时间
                                                              • CMD：启动进程的完整命令

                                                             

                                                            docker inspect {name/Id}

                                                            • 作用：查看镜像、容器、网络、卷的底层详细信息

                                                            • 回显说明：

                                                              • 镜像

                                                                • RepoTags：镜像标签
                                                                • Size：镜像大小
                                                                • Config：镜像默认配置
                                                                • RootFS：镜像分层文件系统信息

                                                              • 容器

                                                                • State：运行状态信息
                                                                • NetworkSettings：网络配置
                                                                • Mounts：挂载信息
                                                                • Config：容器环境变量和命令的配置
                                                                • HostConfig：主机相关配置

                                                             

                                                            4.备份扩容

                                                            docker cp {containerID:containerPath} {hostPath}

                                                            • 作用：用于在容器和本地文件系统之间复制文件/文件夹

                                                             

                                                            docker export {containerID}

                                                            • 作用：将容器的文件系统导出为 tar 归档文件

                                                            • 参数说明：

                                                              • -o：输出到指定文件

                                                            • 示例：

                                                              • docker export mongo > bak.tar
                                                              • docker export -o bak.tar mongo

                                                             

                                                            docker import {imgName:imgTag}

                                                            • 作用：从 tar 归档文件导入内容来创建文件系统镜像

                                                            • 示例：

                                                              • cat bak.tar | docker import - newMongo:myVersion
                                                              • docker import bak.tar newMongo:myVersion

                                                             

                                                            docker commit {containerId} [newImgName:newImgTag]

                                                            • 作用：从容器的当前状态创建一个新的镜像

                                                            • 参数说明：

                                                              • -a：指定作者信息
                                                              • -m：设置提交信息
